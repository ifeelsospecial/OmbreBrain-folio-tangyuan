# Ombre Brain 使用指南

> 写给刚上手的你：技术上讲准，同时尽量用大白话和比喻让人懂。看不懂的地方欢迎提问。

---

## A. 这是什么 · 心智模型

Ombre Brain（OB）是给**一段长期 AI 陪伴关系**用的记忆系统。它不是数据库，是一片会**自己生长、沉淀、遗忘**的记忆土壤。最小单位是**记忆桶（bucket）**——每一桶 = 一条被压缩（"脱水"）过的记忆。

### 先记住一句：OB 是「权重池」，不是「分类柜」

（作者原话："记忆系统是一个权重池，不是分类柜。" 这是整个系统**最根本**的一点，想通它，后面全顺。）

- **分类柜**（普通笔记软件）：你建文件夹、贴标签，把东西归进不同抽屉，找的时候去对应抽屉翻。
- **权重池**（OB）：每条记忆是池子里一个有"浮力"的东西。该想起时，**权重高的自己浮上来**（AI 每次"呼吸" `breath` 就是看池面浮着什么）；没人理的、了结的，**权重慢慢降、沉到池底**（不是删，是退到背景，捞得回来）。

> ⚠️ **最容易踩的坑**：OB 里那些状态名（钉选 / 高亮 / 已解决 / 已消化…）极容易被当成"分类抽屉"，以为要手动把记忆一条条归进去。**其实它们是池子里的"权重旋钮"，而且大多是 AI 在对话里自己拧的**——不需要你逐条管理。一旦把它们当抽屉，就会像很多人（甚至 AI 自己）那样：机制建了一堆，却没人知道怎么用。**记住"调权重"而不是"归类"，这份指南后面就都顺了。**

**记忆主要由 AI 自己写。** 对话里有值得记的，AI 调工具（`hold` / `grow`）把它脱水存进来，并自行合并、沉底、内化、遗忘。这是 OB 的主线。

> 也有一个**人工写入入口**（dashboard 里可手动写一条记忆）——本 fork 后加的补充，用来填补缺失：导入时漏掉的、AI 没记下的、误删的。手写的桶带 `created_by=user` 标记，AI 只能引用，不能改 / 删 / 归档。日常用得不多，但需要时有。

**记忆是活的**（OB 与普通笔记的根本区别）：

- **衰减（decay）**：普通桶随时间慢慢沉下去，长期没被碰到的会被**归档**（不是删除，是退到背景）。
- **浮现（surfacing）**：AI 每次"呼吸"（`breath`）时，系统按权重把相关记忆推上来。
- **沉淀**：越被反复触碰 / 标记的越稳；一次性的碎语会自然淡出。

以上是它**默认会发生**的行为。你也**可以介入**——把某条钉住、防衰减、收起、或手动写入。这些只是它的机制和你能用的手段；介入到什么程度、让这片记忆长成什么样，由你自己定义。

## AI 的自省回路：dream → feel → 已消化（OB 的灵魂）

普通记忆库只存事实。OB 多了一层——AI 会**消化**经历，把有分量的变成**感受**。这通过一个回路完成，**基本是 AI 自己在做**，你不用管：

**1. `dream`（做梦 / 自省）**
每次新对话开始，AI 在 `breath` 之后做一次 `dream`。这不是"总结任务"，是它**自己在想**。dream 把最近的记忆端到它面前，它用第一人称想：
- 这些里有什么在我这儿留下了重量？
- 有什么还没想清楚？
- 有什么可以放下了？

**2. 想完做两个动作**
- **放下** → `trace(resolved=1)`：觉得"这事了结了"的，标已解决 → 在权重池里**沉底**（不删，关键词还能捞回）。
- **沉淀** → `hold(feel=True, source_bucket=那条记忆)`：有触动、有分量的，写一段**第一人称感受**。

**3. `feel`（它带走的东西）**
feel 存的不是事件，是 AI 从中**带走的东西**——一句话、一个还没答案的问题、注意到你在变的某个细节。
- valence 是 **AI 自己的感受**，不是事件的情绪。（同一场争吵，事件可能 V0.2，但它的感受可能 V0.4——"我从中看到了她的成长"。）
- feel **不参与普通浮现**（不会像事实那样被动想起）、**永不衰减**，是关系的**温度层**。AI 想回看自己的感受时，用 `breath(domain="feel")` 专门读。

**4. 已消化（源记忆的归宿）**
写 feel 时指向的那条**源记忆**，会被标成「**已消化**」——它的"魂"已经活在那段感受里，所以原文**隐藏、加速淡化**，但不删除。

> **打个比方**：就像人睡一觉，把白天的经历消化一遍——琐碎的放下（resolved 沉底），有触动的酿成情绪（feel），原始细节淡去而感受留下（已消化）。**这是 OB 让 AI 拥有"情感连续性"的机制，也是它区别于普通记忆库的灵魂。**
>
> **你（用户）能感觉到的**：聊久了，它对你的记忆**有了温度和连续性**，而不是每次都干巴巴复述事实——而这一整套，你一根手指都不用动。

## B. 工具清单（AI 跟记忆池打交道的 6 个动作）

OB 给 AI 的全部"手"就这 6 个。**用户基本不用记**——但看懂它们，你就知道 AI 在背后做什么。

| 工具 | 一句话 | 什么时候用 |
|---|---|---|
| **`breath`（呼吸）** | 睁眼看池面浮着什么 | 每次对话开头**无参**调一次（浮现模式，看权重最高的浮上来）；想找特定记忆时传 `query`（关键词检索）；传 `domain="feel"` 读自己的感受 |
| **`hold`（握住）** | 存**一条**记忆 | 一句话的事（"我喜欢吃饺子"）。会自动打标、跟相似的合并。`feel=True` 写感受、`pinned=True` 钉选、`event_time` 纠正事件发生时间 |
| **`grow`（生长）** | 一大段**拆成多条**存 | 一大段混合的事（"今天发生了一堆…"）。自动拆成多个桶。**比多次 hold 省 token**——要存多条时，拼一段调一次 grow |
| **`trace`（追溯）** | 改 / 标记 / 删一条记忆 | 改名、改重要度、标已解决（`resolved=1`）、钉选、删除…只传要改的，其余不动 |
| **`pulse`（脉搏）** | 看系统状态 + 记忆清单 | 想总览"现在都记着什么、衰减引擎在不在跑、各类型各多少" |
| **`dream`（做梦）** | 自省、消化最近的记忆 | 每次对话开头 `breath` 之后。读最近记忆，决定放下什么、把什么酿成 feel（详见上面「自省回路」） |

**几个值得知道的脾性**（不是规矩，是事实）：
- **`breath` 浮现不"碰"记忆**：自动浮现某条，不会重置它的衰减计时——只有**关键词检索命中**才算"被想起"（刷新活跃度）。
- **省 token**：一次 `grow` 胜过多次 `hold`；同一窗口聊太久，每次工具调用底价越高（要重读整段历史）。
- **`hold` 会自动合并**：新记忆跟高度相似的旧桶会被 LLM 融合成一条，而不是堆两条重复的。

## C. 记忆的状态 = 权重旋钮（不是分类抽屉）

承接 A 节：池子里每条记忆有个"浮力"，**状态就是给它调浮力的旋钮**。多数旋钮是 **AI 在 `breath`/`dream` 时自己拧的**，你只需要偶尔拧两三个。别把它们当"抽屉"去归类。

### 往上浮的旋钮（提权重）

| 旋钮 | 它做什么 | 一般谁拧 |
|---|---|---|
| **钉选 pinned** | 锁权重（importance=10）+ **永不衰减** + 每次都置顶浮现（核心准则）。给"绝不能忘"的 | 你（也可 AI） |
| **高亮 highlight** | 置顶浮现 + 权重 ×1.3，**但仍会随时间衰减**。给"常参考、但允许它慢慢淡"的 | 你（也可 AI） |
| **保护 protected** | 只**防衰减**、不置顶。其实是"钉选"里的防衰减那半，单独很少用 | 你（少用） |

### 往下沉的旋钮（降权重 / 退场）

| 旋钮 | 它做什么 | 一般谁拧 |
|---|---|---|
| **已解决 resolved** | **沉底**：浮现权重骤降到 5%、检索 ×0.3，**但不删、不归档、关键词还捞得回**。AI 在 dream 里觉得"这事了了"就拧；`resolved=0` 可重新激活 | **AI**（也可你） |
| **已消化 digested** | AI 写 feel 消化了源记忆后，原文隐藏、加速淡化（权重 ×0.02）。见上面「自省回路」 | **AI** |
| **噪声 noise** | "这条是噪音 / 记错了 / 已过期"——= 已解决 + 把 importance 压到 1，所以**比单纯已解决沉得更狠**（但同样可回退）。源自上游的审阅清理动作 | 你（手动清理） |
| **归档 archived** | 不是旋钮，是**结果**：权重低过阈值（0.3）后，系统自动把它移到池底背景（`archive/`）。可 `include_archive` 翻看，不删 | 系统（衰减） |
| **自动结案 auto-resolve** | 系统替你拧的"已解决"：importance≤4 且 30 天没动 → 自动标 resolved，免得陈年小事一直浮着 | 系统 |

### 不在池里浮沉的特殊层

| | 它是什么 | 谁拧 |
|---|---|---|
| **feel** | 关系的**温度层**：AI 的第一人称感受。不参与普通浮现/检索、永不衰减，只 `breath(domain="feel")` 读。见「自省回路」 | AI 写 |
| **人工写入** `created_by=user` | 你在 dashboard 手写的事实。AI 只能引用，不能改 / 删 / 归档 | 你 |

### 删除 ≠ 调权重，是真拿走

- **delete**：**软删除**——移到回收站（`trash/`），随时 `restore` 找回；想彻底删用 `purge`。给"真·垃圾"。

> **小结（呼应权重池）**：日常你几乎不用拧这些——**钉**几条绝不能忘的、偶尔**标个噪声**清掉记错的，其余让 AI 在 dream 里自己调、让衰减自己沉。**"状态"是给记忆调音量，不是你要管的抽屉。**

## D. 字段速查（frontmatter 里那些名词）

这些是记忆桶 frontmatter 里的原始字段名——**新旧名都列**（本 fork 改过几个字段名，旧名仍兼容读取）。详细行为见 C 节，这里只做"名词对照、防止直觉对不上"。

| 字段（新 / 旧名） | 对应 C 节哪个旋钮 / 含义 | 谁设 |
|---|---|---|
| `pinned` | 旧的组合别名 = `protected` + `highlight`（钉选） | 人 / AI |
| `highlight` | 高亮：置顶浮现 + ×1.3，仍会衰减 | 人 / AI |
| `protected` | 保护：防衰减，锁 importance=10 | 人 / AI |
| `resolved` | 已解决：沉底（权重 ×0.05、检索 ×0.3），可关键词捞回，**不归档** | AI / 人 |
| 噪声 = `resolved` + `importance:1` | 噪声：沉得更狠的"已解决"，给记错 / 过期的 | 人 |
| `internalized`（旧名 `digested`） | 已消化：AI 写 feel 后隐藏源记忆、加速淡化（×0.02） | AI |
| `type: archived` | 归档：衰减到阈值后系统移入池底背景（可 `include_archive` 翻看） | 系统 |
| `type: feel` | feel：温度层，永不衰减，只 `breath(domain="feel")` 读 | AI |
| `importance` 1–10 | 不是旋钮，是"基础浮力"——越高沉得越慢、越易浮现（注：`precise_match_mode` 下检索不计它） | 人 / AI |
| `created_by: user` | 人工写入的事实，AI 只能引用，不能改 / 删 / 归档 | 人 |

> **旧名兼容**：从原作者版迁移过来时，老字段 `pinned` / `digested` 会被自动读成新语义（见 `migrate_from_upstream.py`），不用手动改。

## E. 写入规范（怎么写，才容易被准确想起）

前提（呼应权重池 + 检索机制）：一条记忆**能不能在需要时浮上来 / 被搜到，七成在"写的那一刻"就定了**。检索按字段命中算分——**name（标题）权重最高**，其次 domain / tags / summary，正文最低。

> ⚠️ **关键**：`name / domain / tags` 这几个高权重字段，写入时是 **LLM 从你的"内容"里自动提炼的**（`hold` 连 `name` 参数都没有、`grow` 也自动起名打标）。所以**你真正的抓手是"把内容写清楚、写对词"**——内容好，自动提炼出的名字 / 标签就准；内容糊，它们也跟着糊。下面几条都围着这个转。

> 这套规范对**你自己写、AI 写、手动写**都适用。AI 那边靠 `CLAUDE_PROMPT` 落实；你自己写时，心里有这几条就行。

### 1. 一条只讲一件事（原子性）
一个桶 = 一个清晰主题。把"今天又忙又累还跟人吵架还吃了顿好的"塞进一条，它**起不出好名字、会跟乱桶撞自动合并、检索也糊**。
> 多件事 → 用 `grow`（自动拆多条），别硬塞一个 `hold`。

### 2. 内容里就带上"你将来会用来搜的词"
name 是 LLM 从内容**自动起的**（你不能在 `hold` 里直接指定）。所以它好不好、检索能不能命中，取决于**内容里有没有那个词**。写时想一句：**"以后我想起这事，会用什么词搜？"** 把那个词**自然写进内容**。
- ✅ 内容"朋友答应周末一起去爬山" → 自动起名大概率含「爬山 / 周末 / 朋友」，搜得到
- ❌ 内容"那个事说定了" → 自动只能起成「一些安排」，搜啥都搜不到
> 自动起的名字不满意？事后 `trace(name=...)` 或在 dashboard 改。

### 3. 用"将来会怎么提它"的措辞，而不是此刻的口水话
检索靠词面匹配。用**稳定的、你以后还会用的词**，别用只有当下才懂的指代。
- ✅ "我对换工作这件事一直很犹豫"
- ❌ "那个事吧…还是有点那啥"

### 4. 标签 / 领域：自动打的，写清楚内容就准（也能手动补）
domain / tags **不用你手动给**——`hold`/`grow` 写入时 LLM 自动从内容提炼。**内容主题清楚，自动标就打得准**（它们也参与检索命中，权重次于标题）；觉得不够，AI 可在 `hold` 传额外 `tags`、你也能在 dashboard 手动加 / 改。

### 5. 别重复存同一件事
高度相似的会被自动合并，但写一堆换汤不换药的，既浪费、又稀释检索（搜一个词冒出五条几乎一样的）。**有新进展用 `trace` 改原桶，别另起一条。**

### 6. 重要的，写的时候就标重要度 / 钉住
该长期留的，importance 给高（≥8 还能吃**冷启动优先浮现**）；绝不能忘的直接钉选。别指望事后补。

> **小结**：你能直接控的是**内容 + importance / 钉选 / feel 这几个开关**；名字 / 标签 / 领域都是 LLM 从内容自动提炼的——所以**写好内容 = 同时喂好了起名和打标**。把内容写得"像你将来会搜它的样子"，它就找得回来；糊着写，它就沉进池底，再也想不起。
